feat: Добавлена система внутренних объектов для учёта расходов/доходов по компании

Реализована возможность создания виртуальных объектов (HolyCowPhuket внутренний объект и филиалы),
которые не удаляются при синхронизации с Beds24 и используются для учёта
расходов/доходов по организации в целом.

Ключевые изменения:

Новые функции:
- Отдельная коллекция MongoDB `internalObjects` для внутренних объектов
- API для инициализации объекта "HolyCowPhuket внутренний объект" и управления филиалами
- Страница администратора для управления внутренними объектами
- Автоматическое объединение объектов из Beds24 и внутренних объектов

Архитектура:
- Использование отрицательных ID (-1, -2, ...) для внутренних объектов
- Отсутствие конфликтов с положительными ID из Beds24
- Идемпотентная инициализация объекта "HolyCowPhuket внутренний объект"
- Сохранение внутренних объектов при синхронизации с Beds24

Новые файлы:
- src/lib/server/internalObjects.ts - управление внутренними объектами
- src/app/api/internalObjects/init/route.ts - API инициализации
- src/app/api/internalObjects/branches/route.ts - API управления филиалами
- src/app/dashboard/internalObjects/page.tsx - UI страница администратора
- INTERNAL_OBJECTS.md - полная документация
- INTERNAL_OBJECTS_QUICK_START.md - быстрый старт
- CHANGES_SUMMARY.md - детальное резюме изменений

Модифицированные файлы:
- src/lib/server/getObjects.ts - объединение objects + internalObjects
- src/app/api/objects/route.ts - поддержка отрицательных ID
- src/app/api/analytics/route.ts - поддержка внутренних объектов
- src/app/api/bysuness/route.ts - поиск в обеих коллекциях
- src/components/leftMenu/MiniDrawer.tsx - новый пункт меню
- src/i18n/translations/ru.json - русские переводы
- src/i18n/translations/en.json - английские переводы

Права доступа:
- Управление внутренними объектами: только admin
- Создание расходов/доходов для компании: admin, accountant
- Просмотр: все авторизованные пользователи

Использование:
1. Администратор инициализирует объект "HolyCowPhuket внутренний объект" через UI или API
2. При необходимости добавляются филиалы компании
3. При создании расходов/доходов объект "HolyCowPhuket внутренний объект" доступен в списке
4. Объект "HolyCowPhuket внутренний объект" не удаляется при синхронизации с Beds24

Документация: см. INTERNAL_OBJECTS_QUICK_START.md для быстрого старта
